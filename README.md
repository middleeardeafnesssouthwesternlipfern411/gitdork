## 🧭 How to Use

1. Open gitdork.
2. Pick the search source you want to use.
3. Choose a target type or a search pattern.
4. Enter a keyword, domain, company name, or term you want to check.
5. Copy the generated dork.
6. Paste it into Google, Shodan, or GitHub search.

### Example uses

- Search for public login pages
- Look for exposed config files
- Find code that contains secret-looking strings
- Check for open services on a domain
- Build repeated searches for a target name

## 🔍 Search Types

### Google dorks

Use these for public web search. They can help you look for:

- File types
- Login pages
- Public documents
- Exposed paths
- Specific words in page titles or URLs

### Shodan dorks

Use these for internet device search. They can help you look for:

- Open ports
- Service names
- Host details
- Web apps
- Certificates and banners

### GitHub dorks

Use these for code search. They can help you look for:

- API keys
- File names
- Config files
- Project names
- Code text matches

## 🛠️ Basic Workflow

1. Start with a target name or domain.
2. Choose the platform you want to search.
3. Add one or two search terms.
4. Copy the generated query.
5. Run the query in the right service.
6. Review the results and refine the terms if needed.

Keep searches small at first. Then narrow them down step by step.

## 📌 Tips for Better Results

- Use short target names first
- Try one query at a time
- Change the file type or keyword when results are too broad
- Keep a list of searches that work well
- Use the same naming style across different targets

Good search terms often include:

- file
- login
- admin
- config
- backup
- token
- key
- password
- api
